ProseMirror is an in-browser semantic rich text editor component. This is what it might look like:

Control your Content

There is something of a gap between MarkDown textareas (limited, hard to use) and classical HTML WYSIWYG editors (content becomes a non-semantic blob). ProseMirror aims to fill this gap.

It does this by decoupling its content model from HTML. Documents are represented with a JavaScript data structure, and their structure is described by a schema that you can provide yourself. This gives you full control over the kind of elements that may appear in your document, and the way they nest and relate to each other.

  • Users get a clean, consistent interface that allows them to enter only documents that fit the schema.
  • You get clean, consistent content that you can format or analyze.

ProseMirror is open-source under an MIT license.

Features

ProseMirror is designed to allow customization and extension throughout.

  • Support for collaborative editing (details)
  • Export and import from ProseMirror's document format to HTML (or Markdown)
  • A clean, immutable document data structure (not the DOM) to program against
  • Document schemas that can be modified and extended without having to rewrite the whole editor
  • A rich set of editing commands that can appear in the menu or be bound to keys, and the ability to define your own
  • An API that easily integrates with React-style or linear dataflow based architectures
  • Menus and other UI elements live outside of the core library, and can be implemented according to your own preference (and the frameworks you may use)
  • A modular architecture to make sure you only load the code you need
  • A plugin system that allows you to easily enable additional functionality and package up your extensions in a convenient format
  • Changes are first-class values and can be stored, inspected, and manipulated

Project

ProseMirror is open source, and you are legally free to use it commercially. Yet, writing, maintaining, supporting, and setting up infrastructure for such a project takes a lot of work and energy. Therefore...

If you are using ProseMirror to make profit, there is a social expectation that you help fund its maintenance.

ProseMirror is being developed on GitHub. Contributions are welcome.

The easiest way to install ProseMirror is with npm. It is currently in beta stage, meaning that you are encouraged to try it out and report issues.

Discussing the project, or asking questions, is best done on the forum. Bugs should be reported through the issue tracker. We aim to be an inclusive, welcoming community. To make that explicit, we have a code of conduct that applies to communication around the project.

Initial development of the software was crowd-funded by these 414 excellent individuals and organizations.
